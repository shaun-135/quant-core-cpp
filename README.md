## 1. 動機與目標

在量化交易領域，系統執行速度與資料存取效率直接決定了策略開發的成敗。面對海量的歷史交易細節（Tick Data），若僅使用基礎陣列進行線性運算，其 $O(N^2)$ 的複雜度將導致回測效率極其低下。

本專案目標在於捨棄高階金融套件，由底層手動實作核心邏輯，建立一個以 **C++** 為基礎的高效能回測引擎。透過精準的資料結構設計，將時序資料的指標運算與檢索效率最佳化，旨在理解電腦在處理大規模序列資料時的效能瓶頸，並產出具備實務邏輯的技術報告。

---

## 2. 預期功能

* **資料注入 (Data Ingestion)**：解析 CSV 格式的歷史股價資料（日期、開高低收、成交量），並將其轉換為記憶體中高效的資料結構物件。
* **指標計算引擎 (Indicator Engine)**：實作技術指標計算模組，包含移動平均線 (MA) 與相對強弱指標 (RSI)。
* **策略回測系統 (Backtesting System)**：模擬買賣邏輯（如均線交叉策略），並即時更新帳戶餘額、持倉部位與成本。
* **績效評估報告 (Performance Analytics)**：自動計算總報酬率 (ROI)、勝率及最大回撤 (Max Drawdown)，並輸出交易日誌。

## 5. 與課程的關聯

* **Circular Buffer (環形緩衝區)**：用於計算滑動視窗指標，將移動平均線的運算複雜度從 $O(N \times W)$ 降至 $O(N)$，實現 $O(1)$ 的即時更新。
* **Hash Table (雜湊表)**：用於多標的管理與部位檢索，確保在 $O(1)$ 時間內完成資產狀態查詢。
* **Binary Search Tree (二元搜尋樹)**：用於儲存交易日誌，支援按時間區間（Range Query）進行快速檢索，複雜度為 $O(\log N)$。
* **Memory Management**：手動管理節點與物件生命週期，深入理解資料結構在 C++ 底層分配的實際影響。
